## 🧭 Overview

This project automates batch monitoring in SAP, reducing the need for recurring manual checks and accelerating communication between Specialists and Production.

The solution reads batches from an Excel file, checks their status directly in SAP and, once a batch arrival at the plant is identified, updates the control spreadsheet and sends automatic alerts through Outlook and Microsoft Teams.

---

## 🧩 Business Problem

In a production environment with a high volume of batches, monitoring depended on manual SAP checks throughout the day. The Specialist had to verify whether a batch had arrived at the plant and only then notify Production to start the inspection steps.

When this identification happened hours after the actual batch arrival, there was a risk of delaying the start of sample collection, inspection, Quality analysis and the handling of potential deviations.

---

## 💡 Implemented Solution

<table>
  <tr>
    <td width="33%">
      <h3>📥 Data Input</h3>
      <p>Automatic reading of batches listed in an Excel database.</p>

    <td width="33%">
      <h3>🤖 RPA Automation</h3>
      <p>Automated execution of SAP queries to validate the status of each batch.</p>
    </td>
    <td width="33%">
      <h3>📢 Communication</h3>
      <p>Simultaneous notifications via Outlook and Microsoft Teams to the responsible areas.</p>
    </td>
  </tr>

The automation was developed in Python using an RPA approach to connect Excel, SAP, Outlook and Microsoft Teams in an end-to-end automated workflow.

The bot reads the batches from the spreadsheet, checks each item in SAP, identifies the arrival at the plant, updates the control database and triggers automatic alerts to Specialists and Production.
